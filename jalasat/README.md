# سامانه جلسات و صورتجلسات ادب‌ویزا

سامانه‌ی داخلی مدیریت درخواست جلسات، بررسی تایم خالی پرسنل، صورتجلسه‌سازی، مصوبات/تسک با گردش‌کار تایید،
اعلان چندمرحله‌ای (پنل + پیامک فراپیامک)، پروفایل پرسنلی و داشبورد گزارش‌گیری.

## پشته‌ی فنی

- **Next.js 14** (App Router) + **TypeScript** + **Tailwind CSS**
- **Prisma ORM** روی **SQLite** برای توسعه (به‌راحتی قابل تغییر به MySQL/Postgres برای سرور)
- **NextAuth** برای احراز هویت (ورود با شماره موبایل و رمز عبور)
- **Recharts** برای نمودارهای داشبورد
- **xlsx (SheetJS)** برای خروجی اکسل آنی (بدون ذخیره فایل روی سرور)
- **jalaali-js** برای تبدیل تاریخ شمسی/میلادی، کاملا آفلاین و لوکال
- **node-cron** برای ورکر یادآوری‌های زمان‌بندی‌شده

معماری کاملا **ماژولار و لایه‌ای** است:

```
src/
  app/            → صفحات و API Routes (لایه‌ی ارائه)
  modules/        → منطق تجاری هر دامنه (meetings, minutes, tasks, notifications, units, reports)
  components/     → کامپوننت‌های UI (ui پایه، layout، هر ماژول کامپوننت‌های خودش)
  lib/            → ابزارهای مشترک (db، auth، jalali، permissions، constants)
```

## پیش‌نیازها

- Node.js نسخه ۱۸ یا بالاتر
- npm

## راه‌اندازی (روی سیستم شما / VS Code)

```bash
# ۱. نصب پکیج‌ها
npm install

# ۲. کپی فایل نمونه‌ی متغیرهای محیطی
cp .env.example .env
# سپس فایل .env را باز کرده و NEXTAUTH_SECRET را با یک رشته‌ی تصادفی طولانی جایگزین کنید

# ۳. ساخت دیتابیس (SQLite) و اجرای مایگریشن
npm run db:push

# ۴. افزودن داده‌های نمونه (واحدها + چند کاربر آزمایشی)
npm run db:seed

# ۵. اجرای پروژه در حالت توسعه
npm run dev
```

سپس آدرس `http://localhost:3000` را باز کنید.

### ورود آزمایشی

بعد از `db:seed`، با هر یک از شماره‌های زیر و رمز عبور `Admin@12345` وارد شوید:

| نقش | شماره موبایل |
|---|---|
| مدیرکل (دسترسی کامل) | `09120000001` |
| مدیرعامل | `09120000002` |
| قائم مقام | `09120000003` |
| مدیر واحد مارکتینگ | `09120000004` |
| کارشناس مارکتینگ | `09120000006` |

## فونت لوکال (بدون CDN)

طبق درخواست، هیچ فراخوانی به CDN بیرونی (نه گوگل‌فونت و نه CDN دیگر) وجود ندارد. قبل از اجرا،
راهنمای `public/fonts/README.md` را دنبال کنید و فایل‌های فونت وزیرمتن را دانلود و در همان مسیر
قرار دهید. تا قبل از افزودن فایل‌ها، سیستم به‌صورت خودکار از فونت Tahoma سیستم‌عامل استفاده می‌کند
تا چیزی خراب نشود.

## اتصال پیامک فراپیامک

در فایل `.env` مقادیر زیر را کامل کنید (مطابق پنل فراپیامک خودتان):

```
FARAPAYAMAK_USERNAME="..."
FARAPAYAMAK_PASSWORD="..."
FARAPAYAMAK_SENDER_NUMBER="..."
FARAPAYAMAK_ENABLED="true"
```

پیاده‌سازی در `src/modules/notifications/providers/sms/farapayamak.provider.ts` طبق مستندات رسمی
وب‌سرویس فراپیامک (متد SendSimpleSMS) نوشته شده. **چون این پروژه در محیطی بدون دسترسی اینترنت ساخته
شده، این بخش تست ارسال زنده نشده است** — پیشنهاد می‌شود قبل از استفاده در محیط عملیاتی، یک بار طبق
مستندات فعلی پنل خودتان (بخش «وب‌سرویس» در پنل فراپیامک) دقیقا تطبیق دهید. تا زمانی که
`FARAPAYAMAK_ENABLED=false` باشد، سیستم به‌جای ارسال واقعی فقط در کنسول لاگ می‌زند (حالت توسعه) و
بقیه‌ی سیستم بدون خطا کار می‌کند.

## یادآوری خودکار جلسات (روز قبل / ۱ ساعت قبل / ۵ دقیقه قبل)

دو روش برای اجرای دوره‌ای این بررسی وجود دارد:

**روش ۱ (پیشنهادی) — ورکر داخلی با pm2 روی سرور:**
```bash
npm run build
pm2 start "npm run notify:worker" --name adabvisa-notifier
pm2 start "npm run start" --name adabvisa-web
```

**روش ۲ — کرون سیستم‌عامل / سرویس‌های بیرونی مثل cron-job.org:**
هر ۱ تا ۲ دقیقه یک‌بار درخواست زیر را بزنید:
```bash
curl -X POST https://your-domain.com/api/notifications/cron \
  -H "x-cron-secret: <مقدار NOTIFICATION_CRON_SECRET در .env>"
```

## انتقال از SQLite به MySQL/Postgres روی سرور

در `prisma/schema.prisma`:
```prisma
datasource db {
  provider = "mysql"   // یا "postgresql"
  url      = env("DATABASE_URL")
}
```
و در `.env`:
```
DATABASE_URL="mysql://user:pass@host:3306/adabvisa_meetings"
```
سپس:
```bash
npm run db:push
npm run db:seed   # اختیاری، فقط برای داده‌ی اولیه
```

## نقش‌ها و سطوح دسترسی

| نقش | دسترسی |
|---|---|
| مدیرکل | دسترسی کامل به همه‌چیز، از جمله تنظیمات سیستم |
| مدیرعامل / قائم مقام | اساین جلسه/تسک به هرکسی در کل سازمان + گزارش کل سازمان |
| مدیر واحد / سرپرست | اساین جلسه/تسک فقط در محدوده‌ی واحد خودشان |
| کارمند | فقط تسک‌ها و جلسات مرتبط با خودشان |

مدیریت واحدها و پرسنل فقط برای «مدیرکل» و «مدیرعامل» در دسترس است.

## سیستم امتیازدهی

طبق درخواست فعلا این بخش فعال نشده، اما زیرساخت آن (مدل `ScoreEvent` در `prisma/schema.prisma`)
از قبل آماده است تا هر زمان خواستید، به‌سادگی قوانین امتیازدهی (مثلا +۵ برای تکمیل به‌موقع تسک،
-۳ برای تاخیر) را در `src/modules/tasks/task.service.ts` اضافه کنید.

## نکات مهم قبل از بردن به محیط عملیاتی

1. `NEXTAUTH_SECRET` و `NOTIFICATION_CRON_SECRET` را حتما با مقادیر تصادفی و امن جایگزین کنید.
2. رمزهای عبور پیش‌فرض کاربران seed را در محیط واقعی تغییر دهید یا کاربران seed را حذف کنید.
3. فایل‌های فونت را طبق `public/fonts/README.md` اضافه کنید.
4. اتصال فراپیامک را طبق مستندات به‌روز پنل خودتان یک‌بار تست کنید.
5. برای آپلود واقعی عکس پروفایل، در حال حاضر فقط وارد کردن URL تصویر پشتیبانی می‌شود؛ برای آپلود
   فایل مستقیم می‌توانید یک endpoint آپلود (مثلا با ذخیره روی دیسک سرور یا S3) اضافه و در
   `src/components/profile/ProfileEditForm.tsx` وصل کنید.
6. تاریخ‌های شمسی در فرم‌ها به‌صورت متنی با فرمت `۱۴۰۴/۰۳/۰۱` وارد می‌شوند؛ در صورت نیاز می‌توانید
   یک date-picker گرافیکی شمسی (مثلا کتابخانه‌ی `react-multi-date-picker`) جایگزین این ورودی‌های
   متنی کنید — تمام منطق تبدیل تاریخ در `src/lib/jalali.ts` متمرکز است.
